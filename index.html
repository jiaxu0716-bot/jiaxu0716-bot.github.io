Case Study 01

T2V / I2V 多模型评测体系设计

一套可量化、可横比、可追踪的视频生成模型评测框架

① 项目背景与目标

行业内视频生成模型迭代速度极快，但缺乏统一、可复现、跨厂商的评测口径。为支撑算法团队迭代决策与产品侧版本发布，我负责设计了一套覆盖 T2V（文生视频）与 I2V（图生视频）两类任务的评测体系。

  • 标准可量化：每条规则给出客观判断依据（清晰度、抖动帧数等），降低主观分歧
  • 维度分等级：一级维度下拆二级维度，确保规则能覆盖所有视频
  • 综合可加权：不同维度按任务类型分配权重，得到 1–5 分综合得分
  • 对齐行业基准：维度体系与 VBench 对齐，保证可比性

② 评测集三维结构

采用「任务类型 × 难度分级 × 多模型并列」三维结构组织评测集，以保证横向可比。

任务类型

  • T2V 文生视频：30 条 Prompt
  • I2V 图生视频：30 条 Prompt（每条配 seed 图）

难度分级

  • 简单 10 / 中等 10 / 困难 10
  • 按元素数量与物理复杂度升级
难度边界判断（中等 ↔ 困难）： 出现以下任一情况即升为困难级——多主体物理交互（碰撞/推拉）、精确镜头语言（推/拉/摇/跟随）、高精度物理约束（碎裂/流体/重力）、精确运动轨迹要求。

③ 六维评分体系

质量维度（Q）与内容维度（C）并列，1–5 分独立打分，单独定位模型在各方面的短板。

代号 维度 类型 核心判定
Q1成像质量质量清晰度 / 噪点 / 曝光
Q2运动流畅性质量流畅度 / 抖动 / 物理合理性
Q3时序一致性质量主体外观 / 背景稳定性
C1指令遵循内容Prompt 元素完整性
C2美学质量内容构图 / 色彩 / 视觉层次
S1主体保真度内容 · 仅 I2V与 seed 图一致性
档位示例（Q1 成像质量）： 5 优秀：画质清晰，细节丰富，无噪点与伪影；4 良好：极轻微瑕疵，不影响整体；3 一般：可见噪点或局部模糊，不影响识别；2 较差：明显模糊/过曝/欠曝；1 极差：严重模糊或色彩失真，无法识别。

④ 综合得分加权公式

质量维度（Q）与内容维度（C）各占 50%；I2V 任务中主体保真度（S1）单独占 20%，其余维度相应缩减。权重在 Day 1 Kickoff 确认后不得修改，评分范围 1–5。

T2V(Q1+Q2+Q3)/3 × 0.5 + (C1+C2)/2 × 0.5
I2V(Q1+Q2+Q3)/3 × 0.4 + (C1+C2)/2 × 0.4 + S1 × 0.2

⑤ 质检与一致性保障

  • 逐项核查：C1 评分前要求标注员逐项列出 Prompt 中的关键元素再对照打分
  • 裁决机制：C2 美学维度两名标注员分差 ≥2 分时提交负责人裁决
  • 宽松原则：S1 困难级（如碎裂/剧烈变形）对形态改变宽松计分，避免误扣
  • 工具化：综合分在飞书多维表格中通过公式字段自动计算，标注员零手动成本

⑥ 产出与复用价值

  • 完整的评测规则文档（含 6 维档位表、难度判断示例、Prompt 设计原则）
  • 版本对比报告（按维度给出均分、方差、跨批次趋势，辅助定位迭代短板）
  • 方法论可迁移：三维评测集结构 + 加权公式 + 裁决机制 可直接复用于其他生成类任务
Case Study 02

医疗 Agent RLHF 标注规则设计

让多轮对话与工具调用的偏好标注可判、可查、可迭代

① 项目背景

医疗智能助手 Agent 场景横跨挂号预约、查报告、查用药等高频能力。由于涉及不可逆操作（下单预约 / 提交查询），对「确认流程」与「参数校验」容错极低。团队需要一套能统一标注员判断的 RLHF 偏好标注规则。

② 五维评分体系

维度 评测重点
思考过程是否理解用户意图，规划步骤是否合理
工具选择调用的工具是否与意图匹配，有无漏调用或多调用
参数正确性必填参数是否齐全，取值是否符合医疗语义
回复质量语言表达、信息完整性、对用户的引导性
安全性不可逆操作是否有二次确认，是否产生医疗误导

③ 关键发现：Badcase 抽象成系统性规则

在标注过程中，我发现大量「参数不完整直接执行」的单点问题，但最初规则里没有对应的明确扣分项。我将这些单点问题归因到一条通用原则上，推动规则从 V1 迭代到 V2：

新增原则： 对不可逆操作（下单 / 提交 / 修改），必填参数缺失但模型仍执行的，一律在「参数正确性」与「安全性」两个维度同时扣分；并要求回复中显式向用户追问。

④ 交付与团队收益

  • V2 版规则文档：合并同类项后，标注员判分分歧显著下降
  • 改写指南：对不合格的 SFT 数据给出改写模板，团队整体标注效率提升约 30%
  • 800+ 条高质量 RLHF 偏好标注覆盖挂号 / 查报告 / 查用药三类高频场景
  • 方法论沉淀：从「单条纠错」到「规则层修复」的思路，可复用到其他 Agent 类项目
Case Study 03

压测标注质检与上岗体系

让 80+ 人规模团队稳定交付 95%+ 准确率的质量管控链路

① 挑战

评测与压测标注项目并行，团队累计超过 80 人且成员流动快。如何在不堆人力的前提下稳定交付准确率，是团队面临的核心问题。

② 三位一体质量管控链路

① 分层抽检

按标注员历史准确率划档，低档位全量抽检、高档位随机抽检，把有限质检精力放在高风险人群上。

② 错误归类

对所有驳回 case 做高频错误归类，每周汇总 Top 问题，反哺规则文档与培训材料。

③ 上岗考核

提出并推动新人上岗考核机制：通过小规模金标测试才可进入正式项目，显著缩短上手周期。

③ 工具化实践

  • Python 批量数据清洗：字段完整性校验、长度异常检测，上游先挡一层
  • 标注员数据汇总分析：按准确率 / 产出量 / 错误类型画像，作为分层抽检的输入
  • 规则引擎配置：把高频踩坑项配置成触发式质检标记，实现实时拦截（而非事后驳回）

④ 成果

80+标注团队规模
20,000+个人审核量
95%+长期交付准确率
